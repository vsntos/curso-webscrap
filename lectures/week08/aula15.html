O que é o Projeto Final

Construir um dataset original

Um projeto de pesquisa completo que envolve:

  • pergunta de pesquisa clara;
  • coleta de dados originais;
  • limpeza e documentação;
  • análise substantiva;
  • interpretação teórica.

O que não é

  • reprodução de análise já publicada;
  • download de dados prontos;
  • análise sem pergunta teórica.

Estrutura do Projeto

Cinco etapas obrigatórias

  1. Pergunta — o que você quer saber?
  2. Coleta — de onde e como?
  3. Limpeza — o dado está pronto para análise?
  4. Análise — o que os dados mostram?
  5. Interpretação — o que isso significa para RI?

Etapa 1: Pergunta de Pesquisa

Uma boa pergunta de pesquisa

  • é específica (não “como funciona a diplomacia”);
  • é empírica (pode ser respondida com dados);
  • tem variação (há diferença a ser explicada);
  • tem relevância teórica (conecta com literatura).

Exemplos

  • Como a cobertura brasileira do conflito ucraniano difere da cobertura europeia?
  • Sanções econômicas afetaram as exportações russas pós-2022?
  • Quais temas dominam os discursos do Itamaraty no G20?
  • A presença do Brasil nos fóruns multilaterais cresceu após 2023?

Critérios de escolha

  • robôs.txt permite scraping?
  • há API disponível (prefira)?
  • qual período coberto?
  • qual idioma?
  • qual granularidade?

Princípios

# 5. Sempre registrar metadados de coleta

Estrutura de coleta

source("scripts/utils/scraper.R")

df_raw <- coletar_corpus(
  portal      = "Agência Brasil",
  url_base    = "https://agenciabrasil.ebc.com.br/internacional?page=",
  n_paginas   = 20,
  delay       = 1.5

# Salvar imediatamente
write_csv(df_raw, paste0(

Etapa 4: Limpeza

Checklist de limpeza

Pipeline de limpeza

df_limpo <- df_raw |>
  mutate(
    titulo    = str_squish(titulo),
    data      = lubridate::ymd(data_coleta),
    n_words   = stringr::str_count(titulo, "\\S+")
  ) |>
  filter(
    nchar(titulo) > 20,
    !is.na(data)
  ) |>
  distinct(link, .keep_all = TRUE)

Etapa 5: Análise

O mínimo esperado

  • estatísticas descritivas do corpus;
  • frequência de palavras-chave;
  • variação por portal / período;
  • pelo menos uma análise temática ou de framing.

O que surpreende

  • descobertas contra-intuitivas;
  • padrões que a teoria não previa;
  • diferenças entre fontes;
  • mudanças ao longo do tempo.

library(targets)
library(tarchetypes)

source("scripts/utils/scraper.R")

list(
  # Coleta
  tar_target(raw_agencia, coletar_agencia(0:19)),
  tar_target(raw_onu,     coletar_onu(0:9)),

  # Integração
  tar_target(corpus_bruto, bind_rows(raw_agencia, raw_onu)),

  # Limpeza
  tar_target(corpus_limpo, limpar_corpus(corpus_bruto)),

  # NLP
  tar_target(dfm_corp, construir_dfm(corpus_limpo)),
  tar_target(keyness,  calcular_keyness(dfm_corp)),
  tar_target(frames,   aplicar_frames(dfm_corp)),

  # Outputs
  tar_target(fig_freq,    plotar_frequencia(dfm_corp), format = "file"),
  tar_target(fig_keyness, plotar_keyness(keyness),    format = "file"),
  tar_target(tab_frames,  exportar_tabela(frames),    format = "file"),

  # Relatório
  tar_quarto(relatorio, "docs/relatorio_final.qmd")

Limitações e Validade

Seção obrigatória

Todo projeto deve discutir:

  • viés de seleção — o dado representa o quê?
  • validade de medida — o indicador mede o conceito?
  • temporalidade — quando foi coletado?
  • representatividade — qual universo?
  • fragmentação — o que não conseguimos capturar?
