Pensar como Pesquisador

Não apenas como programador

Código que funciona hoje pode:

  • ser incompreensível em 6 meses;
  • depender de pacotes descontinuados;
  • produzir resultados diferentes ao re-executar;
  • ser impossível de auditar.

Reprodutibilidade é padrão científico

Pesquisa computacional deve ser:

  • reproduzível — outros conseguem re-executar;
  • auditável — outros conseguem verificar;
  • documentada — outros entendem o que foi feito.

Estrutura de Pastas

Projeto padrão de scraping em RI

meu_projeto/
├── data/
│   ├── raw/          # dados brutos, nunca editados
│   └── processed/    # dados limpos e prontos
├── scripts/
│   ├── 01_coleta.R
│   ├── 02_limpeza.R
│   └── 03_analise.R
├── outputs/
│   ├── figures/
│   └── tables/
├── docs/
│   └── metodologia.md
├── logs/
│   └── scraping_20260512.txt
├── .Renviron          # variáveis secretas (não vai ao Git)
├── .gitignore
└── README.md

Separação Raw / Processed

Por que nunca modificar raw?

  • raw é o registro histórico da coleta;
  • se algo der errado no processamento, você tem o original;
  • permite reprocessar sem re-raspar;
  • facilita auditoria.

Princípio

raw/   → escrever uma vez, nunca sobrescrever
processed/ → resultado de scripts reproduzíveis

Convenções

  • sem espaços — use _ ou -;
  • tudo em minúsculas;
  • datas em ISO: YYYYMMDD;
  • numeração para scripts sequenciais.

.Renviron: Segredos

Nunca coloque credenciais no código

# NUNCA:
api_key <- "abc123secreto"

# SEMPRE:
api_key <- Sys.getenv("MINHA_API_KEY")

~/.Renviron

# Arquivo em ~/ (home do usuário)
NEWS_API_KEY=abc123secreto
PORTAL_USER=meu_usuario
PORTAL_SENHA=minha_senha
WTO_KEY=chave_aqui
# Recarregar após editar
readRenviron("~/.Renviron")

# Acessar
Sys.getenv("NEWS_API_KEY")

Reprodutibilidade de Pacotes

O problema

Pacotes mudam entre versões.

O que funciona em maio de 2026 pode quebrar em 2027.

renv: lockfile de pacotes

install.packages("renv")

# Iniciar renv no projeto
renv::init()

# Salvar versões atuais
renv::snapshot()

# Restaurar ambiente em outro computador
renv::restore()
